| 瞬き | 回数だけ | **瞬き持続時間の分布**。疲労→遅い瞬き(>200ms)が増える。頻度より持続時間が重要 |

## Layer 2: 潜在状態の推定（眠気レベル）

ここが**逆問題**。観測信号から隠れた状態を推定する。

推定すべきは1つだけ：**眠気レベル（KSS相当: 1-9段階）**

```
KSS (Karolinska Sleepiness Scale)
1: 非常に覚醒  → PERCLOS < 0.02, 瞬き持続時間 < 100ms
5: どちらでもない → PERCLOS 0.02-0.08
7: 眠いが起きていられる → PERCLOS 0.08-0.15, 遅い瞬き増加
9: 非常に眠い、起きていられない → PERCLOS > 0.15, 微睡みあり
```

現状のif/elifの閾値判定はここを**ルックアップテーブル**で解いてる。研究レベルなら：

- **回帰モデル**（PERCLOS + 瞬き持続時間 + SDBI → KSS推定）
- 訓練データ = 被験者にKSS自己評価させながら顔を撮る
- 個人差は**混合効果モデル**か、初回キャリブレーションで個人切片を推定

## Layer 3: 推定睡眠時間 & 寝るべき度（モデルベース推論）

ここが一番難しい。**今のコードは疲労度から線形マッピングしてるだけ**で、これが本質的に間違ってる。

### 推定睡眠時間：Two-Process Model (Borbély, 1982)

眠気は2つのプロセスの合成：

```
眠気(t) = S(t) - C(t)

S(t): 睡眠負債（起きてる時間に比例して蓄積）
  起床中: S(t) = S_max × (1 - e^(-t/τ_rise))   τ_rise ≈ 18.2h
  睡眠中: S(t) = S_min × e^(-t/τ_decay)          τ_decay ≈ 4.2h

C(t): サーカディアンリズム（時刻の関数）
  C(t) = A × cos(2π(t - φ)/24)    φ ≈ 各個人の体内時計ピーク
```

**つまり「推定睡眠時間」を出すには：**
1. Layer 2で眠気レベルDを推定
2. 現在時刻tを取得（サーカディアン位相C(t)を計算）
3. S(t) = D + C(t) で睡眠負債を逆算
4. S(t)から覚醒時間を逆算 → 24h - 覚醒時間 ≈ 睡眠時間

**現状のコードが「徹夜なのに7.2h」と出した根本原因：** 時刻情報を一切使ってない。朝9時に同じ顔でも、深夜3時に同じ顔でも、意味が全然違う。

### 寝るべき度：パフォーマンス低下モデル

Dawson & Reid (1997)の知見：
- 17時間覚醒 ≈ 血中アルコール0.05%
- 24時間覚醒 ≈ 血中アルコール0.10%（泥酔レベル）

```
パフォーマンス低下率(%) = f(覚醒時間, サーカディアン位相)
寝るべき度 = パフォーマンス低下率 × コンテキスト重み
```

「寝るべきか」は測定値じゃなく**意思決定関数**。コンテキスト（コーディング中？運転中？）で閾値が変わる。

---

## まとめ：正しい因数分解

```
カメラ → [PERCLOS, 瞬き持続時間, SDBI, 色差, 頭部動揺]  ← Layer 1: 計測
       → [KSSスコア (1-9)]                              ← Layer 2: 状態推定（回帰モデル）
       → [S(t), 覚醒時間, 推定睡眠]                      ← Layer 3a: Two-Process逆算（+時刻必須）
       → [寝るべき度]                                    ← Layer 3b: 意思決定関数（+コンテキスト）
```

**今のコードで最も効果が高い改善3つ：**

1. **PERCLOSを実装する**（30fpsの動画が必要。imagesnap 10枚では不可能 → OpenCVのVideoCapture権限を取るか、連続撮影の間隔を100msにする）
